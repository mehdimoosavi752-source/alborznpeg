# نوین پلی‌تکنیک — راهنمای اجرا و آپلود روی سرور

این یک پروژه‌ی React ساخته‌شده با Vite است. Tailwind از طریق CDN در `index.html` لود می‌شود، پس نیازی به کانفیگ اضافه نیست.

## ۱) اجرا روی سیستم خودتان (اختیاری، برای تست قبل از آپلود)

نیاز: Node.js نسخه ۱۸ به بالا

```bash
npm install
npm run dev
```

سپس آدرسی که ترمینال نشان می‌دهد (معمولاً `http://localhost:5173`) را در مرورگر باز کنید.

برای ساخت نسخه‌ی نهایی (production build):

```bash
npm run build
```

خروجی در پوشه‌ی `dist` ساخته می‌شود.

## ۲) آپلود روی GitHub

```bash
git init
git add .
git commit -m "نوین پلی‌تکنیک - نسخه اولیه"
git branch -M main
git remote add origin https://github.com/USERNAME/REPO-NAME.git
git push -u origin main
```

جای `USERNAME/REPO-NAME` را با مخزن (repository) خودتان روی گیت‌هاب جایگزین کنید.

## ۳) دیپلوی روی هاست

این پروژه یک **سایت استاتیک** است (بعد از build فقط فایل‌های HTML/CSS/JS تولید می‌کند)، بنابراین با هر یک از این سرویس‌ها به‌راحتی و رایگان بالا می‌آید:

### گزینه‌ی پیشنهادی: Vercel یا Netlify
سریع‌ترین و بدون‌دردسرترین گزینه برای پروژه‌ی Vite/React:
1. وارد vercel.com یا netlify.com شوید و با اکانت گیت‌هاب لاگین کنید.
2. مخزن (repo) این پروژه را انتخاب کنید.
3. تنظیمات build به‌صورت خودکار تشخیص داده می‌شود:
   - Build Command: `npm run build`
   - Output Directory: `dist`
4. دکمه‌ی Deploy را بزنید — چند ثانیه بعد سایت روی یک آدرس واقعی بالا می‌آید.

### روی Render
1. وارد render.com شوید → New → **Static Site**
2. مخزن گیت‌هاب را متصل کنید
3. تنظیمات:
   - Build Command: `npm install && npm run build`
   - Publish Directory: `dist`
4. Create Static Site را بزنید.

بعد از دیپلوی، دامنه‌ی اختصاصی خودتان (مثلاً novinpolytechnic.ir) را از بخش Domain همان سرویس به پروژه متصل کنید.

## نکته‌ی خیلی مهم درباره‌ی پنل مدیریت

فایل `src/lib/storage.js` فعلاً داده‌ها (محصولات، صفحات، منو، سفارشات و...) را در **localStorage مرورگر** ذخیره می‌کند. یعنی:

- تغییراتی که شما در پنل مدیریت می‌دهید، فقط در همان مرورگر و همان کامپیوتر ذخیره می‌شود.
- بازدیدکننده‌های دیگر این تغییرات را نمی‌بینند، چون داده به‌صورت مرکزی و مشترک ذخیره نمی‌شود.

برای اینکه پنل مدیریت واقعاً روی یک سرور واقعی کار کند و تغییرات برای همه‌ی بازدیدکننده‌ها یکسان باشد، باید یک دیتابیس واقعی وصل شود؛ ساده‌ترین گزینه‌ها:

- **Supabase** (رایگان، دیتابیس Postgres + API آماده)
- **Firebase** (رایگان، دیتابیس Firestore)
- یا یک بک‌اند اختصاصی (Node.js + Express + دیتابیس) روی همان Render

اگر بخواهید، در مرحله‌ی بعد می‌توانم `storage.js` را به یکی از این گزینه‌ها (مثلاً Supabase) وصل کنم تا پنل مدیریت واقعاً روی سرور و برای همه‌ی کاربران کار کند.

## نکته درباره‌ی درگاه پرداخت

بخش پرداخت فعلاً شبیه‌سازی‌شده است. برای اتصال به یک درگاه واقعی (زرین‌پال، به‌پرداخت ملت و...) نیاز به یک بک‌اند است که کلید API درگاه را نگه دارد و مراحل «درخواست پرداخت» و «تایید پرداخت» را با درگاه رد و بدل کند؛ این کار امن نیست که مستقیم از داخل کد فرانت‌اند (مرورگر) انجام شود.
